Si le pedimos a un LLM que resuelva de golpe un problema lógico o matemático complejo, lo más probable es que falle,. Para solucionar esto sin necesidad de alterar los pesos del modelo, utilizamos la técnica de **Chain-of-Thought (CoT)**, donde obligamos al sistema a razonar paso a paso antes de emitir un resultado. Al pedirle explícitamente que "piense en voz alta" (el clásico *"Let's think step by step"*), el modelo despliega el problema de forma secuencial y aumenta drásticamente su precisión.
<br><br>
Para llevar esta fiabilidad a un nivel de producción, los arquitectos de IA combinan *CoT* con una estrategia llamada **Self-Consistency** (también conocida como *Majority Voting over CoT*). Como los LLMs son de naturaleza probabilística, una misma pregunta puede generar caminos de razonamiento distintos dependiendo de su configuración de temperatura. En lugar de confiar ciegamente en el primer intento, el sistema ejecuta la misma consulta múltiples veces para generar diversas rutas de pensamiento y, al final, selecciona la respuesta que más se repite por votación mayoritaria,.
<br><br>
Acá tenés cómo se ven ambas técnicas implementadas sistémicamente en código usando la API de **Gemini**. Vamos a pedirle al modelo que resuelva un problema, forzando su razonamiento y generando múltiples caminos simultáneos para buscar consistencia:
<br><br>

```python
from dotenv import load_dotenv
from google import genai

# Cargamos las variables de entorno e inicializamos el cliente
load_dotenv()
client = genai.Client()

# Paso 1: Diseñamos el System Prompt para forzar Chain-of-Thought (CoT).
# Le exigimos que escriba sus pensamientos lógicos antes de dar la respuesta final.
system_instruction = """
You are an expert logical reasoner.
Always think step-by-step and write down your reasoning steps before providing the final answer.
End your response strictly with 'FINAL ANSWER: [result]'.
"""

# Definimos el problema lógico para nuestro usuario
user_query = "If John has 3 apples, gives away 1, buys 5 more, and splits them equally with his brother, how many apples does each get?"

# Paso 2: Implementamos Self-Consistency (Majority Voting).
# Subimos un poco la 'temperature' para que el modelo explore lógicas y caminos distintos.
# Ejecutamos el modelo 3 veces en un bucle para obtener múltiples resoluciones.
reasoning_paths = []
for i in range(3):
    interaction = client.interactions.create(
        model="gemini-3.5-flash",
        system_instruction=system_instruction,
        generation_config={"temperature": 0.7},
        input=user_query
    )
    reasoning_paths.append(interaction.output_text)

# Paso 3: Mostramos las rutas de pensamiento generadas por el modelo.
# En un producto real, aquí parsearíamos el texto para extraer el 'FINAL ANSWER'
# de cada ruta y elegiríamos el resultado matemático más frecuente.
for idx, path in enumerate(reasoning_paths):
    print(f"--- Reasoning Path {idx + 1} ---")
    print(path, "\n")
```
