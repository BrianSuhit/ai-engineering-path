Como *AI Engineers*, sabemos que los modelos fundacionales son motores excepcionales para reconocer patrones. Para tareas simples, generalmente basta con darles una instrucción directa sin proporcionarles ningún ejemplo previo, lo que en la industria conocemos como **Zero-Shot Prompting**. Sin embargo, cuando la tarea es compleja o cuando nuestro sistema de software requiere que la salida tenga una estructura o tono estrictamente predecible, explicarle al modelo "qué hacer" no siempre alcanza; tenemos que mostrarle "cómo hacerlo".
<br><br>
A esta técnica se la conoce como **In-Context Learning** (Aprendizaje en Contexto). En lugar de alterar la arquitectura o modificar los pesos matemáticos del modelo mediante un costoso *fine-tuning*, le "enseñamos" el comportamiento deseado inyectando pares de ejemplos (entrada y salida ideal) directamente dentro de su ventana de contexto. Si le pasamos un solo ejemplo, lo llamamos **One-Shot**; si le pasamos varios, estamos aplicando **Few-Shot Prompting**. Al alimentar al modelo con estos patrones antes de hacerle la pregunta real, guiamos su proceso de inferencia de manera casi determinista.
<br><br>
Acá tenés cómo se implementa esta técnica en código usando la API de **Gemini**. Vamos a crear un clasificador de sentimientos estricto enseñándole al modelo con unos pocos ejemplos (*Few-Shot*):
<br><br>

```python
from dotenv import load_dotenv
from google import genai

# Cargamos el entorno e inicializamos el cliente de Gemini
load_dotenv()
client = genai.Client()

# Paso 1: Definimos las reglas inquebrantables en el System Prompt
system_instruction = """
You are a strict sentiment analyzer.
Analyze the review and reply ONLY with 'POSITIVE', 'NEGATIVE', or 'NEUTRAL'.
"""

# Paso 2: Aplicamos In-Context Learning (Few-Shot Prompting).
# Le mostramos al modelo exactamente el formato de entrada y salida que esperamos.
few_shot_examples = """
Review: The product was amazing and delivery was super fast!
Sentiment: POSITIVE

Review: It broke after two days of use. Terrible quality.
Sentiment: NEGATIVE

Review: It works exactly as expected, nothing special to mention.
Sentiment: NEUTRAL
"""

# Paso 3: Preparamos la consulta real del usuario
user_query = "Review: The battery life is decent, but the screen scratches easily.\nSentiment:"

# Paso 4: Ensamblamos el 'machete' de ejemplos junto con la consulta final
full_prompt = f"{few_shot_examples}\n{user_query}"

# Paso 5: Ejecutamos el modelo en la nube inyectando todo el contexto
interaction = client.interactions.create(
    model="gemini-3.5-flash",
    system_instruction=system_instruction,
    input=full_prompt
)

# Imprimimos la respuesta, que gracias a los ejemplos, será estrictamente una palabra
print(interaction.output_text)
```
