Llegó el momento de ensamblar el motor. En producción, un *AI Engineer* rara vez programa la matemática de los vectores a mano. En su lugar, orquestamos el flujo completo de la información usando una base de datos como **ChromaDB**. Esta herramienta abstrae la complejidad: recibe tus textos crudos, invoca automáticamente el modelo de *embeddings* (por defecto usa el **`all-MiniLM-L6-v2`** que vimos en el Nodo 8) para convertirlos en vectores, y los guarda en su índice optimizado.
<br><br>
Para entender este flujo arquitectónico, vamos a simular el cerebro de un agente universitario. Primero, creamos nuestra colección (nuestro "disco duro" semántico). Segundo, inyectamos el conocimiento externo: una lista de las materias que tengo en el segundo cuatrimestre junto con sus metadatos. Finalmente, simulamos la consulta de un usuario (*query*). En lugar de buscar una coincidencia exacta de palabras, el sistema calculará la distancia matemática para devolvernos el contexto correcto, incluso si el usuario no sabe el nombre exacto de la materia.
<br><br>
Este código representa exactamente lo que ocurre milisegundos antes de llamar a un LLM. El resultado de este *script* (el documento recuperado) es el contexto que le pasaremos al modelo de lenguaje en el siguiente nodo para que pueda responder con precisión:
<br><br>

```python
# Primero se importa el paquete de chromadb con 'uv add chromadb'
import chromadb

# 1. Inicializa un cliente de ChromaDB en memoria (nuestro disco duro semántico)
chroma_client = chromadb.Client()

# 2. Crea una Colección (equivalente a una tabla en SQL)
# Por defecto, Chroma utiliza automáticamente el modelo de embedding 'all-MiniLM-L6-v2'
collection = chroma_client.create_collection(name="brian_university_subjects")

# 3. Ingesta documentos y metadatos
# Chroma convierte automáticamente este texto a vectores y los indexa
collection.add(
    documents=[
        "Brian cursa Programación en el 2do cuatrimestre.",
        "Brian cursa Web 1 en el 2do cuatrimestre.",
        "Brian cursa Inglés 2 en el 2do cuatrimestre.",
        "Brian cursa Teoría de la información en las organizaciones en el 2do cuatrimestre."
    ],
    metadatas=[
        {"category": "software"},
        {"category": "software"},
        {"category": "language"},
        {"category": "business"}
    ],
    ids=["subj_prog", "subj_web", "subj_eng", "subj_teo"]
)

# 4. Realiza una Búsqueda de Similitud Semántica
# El usuario no sabe el nombre exacto de la materia, solo el concepto
user_query = "¿Qué materia tiene Brian que sirva para hacer páginas en internet?"

# Buscamos el único vector matemáticamente más cercano (k-NN / ANN)
results = collection.query(
    query_texts=[user_query],
    n_results=1
)

# 5. Muestra el contexto recuperado
print("User Query:", user_query)
print("Retrieved Context:", results['documents'])

```
