En las primeras iteraciones de una aplicación de IA, es común caer en la trampa de utilizar un **Contexto Estático**: el desarrollador inyecta (o "dumpea") todos los manuales, archivos de código y descripciones de herramientas en el *prompt* por adelantado esperando que el modelo encuentre lo que necesita. Aunque ventanas masivas de 2 millones de tokens hacen que esto sea posible, en producción es una pésima decisión arquitectónica. Obligar al modelo a leer 50 documentos irrelevantes en cada turno dispara la latencia, multiplica los costos y satura la atención del LLM.
<br><br>
Para escalar, los arquitectos de IA hacemos la transición hacia un **Contexto Dinámico** aplicando la estrategia de **Just-in-Time Retrieval (Selección)**. En lugar de cargar toda la base de datos de golpe, el sistema mantiene referencias ligeras. Cuando el usuario hace una pregunta, nuestro código intercepta la consulta, busca y carga en la memoria RAM del modelo *únicamente* los fragmentos de datos que son estrictamente relevantes para resolver ese problema exacto en ese momento.
<br><br>
Este cambio de paradigma es el puente directo hacia sistemas avanzados. Pasar de inyectar datos de forma estática a recuperarlos bajo demanda es exactamente lo que da vida a arquitecturas como **RAG** (Retrieval-Augmented Generation) y sienta las bases para los **Agentes Autónomos**, donde el sistema decide dinámicamente qué información necesita ir a buscar antes de emitir una respuesta.
<br><br>
Acá tenés cómo se ve este patrón de diseño en código usando la API de **Gemini**. Simulamos una función de recuperación que carga dinámicamente solo un fragmento útil, en lugar de enviarle al modelo todo un manual de 500 páginas:
<br><br>

```python
import os
from dotenv import load_dotenv
from google import genai

# Cargamos el entorno e inicializamos el cliente
load_dotenv()
client = genai.Client()

system_instruction = "You are a helpful customer support AI. Use the provided context to answer."

# --- JUST-IN-TIME RETRIEVAL (Select) ---
# En el Nodo 8, esta función se conectará a una Base de Datos Vectorial real.
# Por ahora, simulamos que el código busca dinámicamente solo la regla relevante.
def get_dynamic_context(query: str) -> str:
    if "shipping" in query.lower():
        return "Context: Shipping takes 2-3 business days via FedEx."
    elif "refund" in query.lower():
        return "Context: Refunds are processed within 5-7 business days."
    return "Context: No specific information found in the database."

# Paso 1: Recibimos la consulta del usuario
user_query = "Hi! How long does shipping usually take?"

# Paso 2: Recuperamos el contexto dinámico 'Just-in-Time' basado en la consulta
relevant_context = get_dynamic_context(user_query)

# Paso 3: Ensamblamos el prompt final inyectando únicamente el dato filtrado
full_input = f"{relevant_context}\n\nUser Question: {user_query}"

# Paso 4: Ejecutamos el modelo en la nube con un contexto limpio, rápido y barato
interaction = client.interactions.create(
    model="gemini-3.5-flash",
    system_instruction=system_instruction,
    input=full_input
)

print(interaction.output_text)

```
