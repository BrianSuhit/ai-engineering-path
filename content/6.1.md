Como arquitectos de sistemas, ya no nos comunicamos con los LLMs enviando un bloque de texto plano esperando que lo autocompleten. El estándar moderno de la industria divide la entrada de datos en una estructura estricta de mensajes con roles específicos: **System** (el desarrollador), **User** (el usuario final) y **Assistant** (la respuesta del modelo). Esta arquitectura nos permite separar las reglas de negocio inquebrantables de la entrada variable del cliente, garantizando que el modelo mantenga su comportamiento de forma predecible sin confundir quién le está dando las órdenes.
<br><br>
Para que esta comunicación sea verdaderamente efectiva, diseñamos el **System Prompt** inyectando módulos clave: una **Persona** (qué identidad o rol debe asumir), el **Context** (la información de fondo o reglas del juego), un **Tone** y un **Format** de salida. Recién cuando el "cerebro" está configurado con nuestra arquitectura, le pasamos la tarea específica a través del **User Prompt**. Cada vez que el modelo procesa la información y genera una respuesta, esa salida se etiqueta bajo el rol de **Assistant**, creando un historial estructurado que el sistema puede almacenar como memoria para mantener el contexto de la conversación.
<br><br>
Acá tenés cómo se ve esta anatomía en código real usando el SDK moderno de **Gemini**, estructurando las reglas del sistema y enviando la consulta para ejecutar la inferencia en la nube:
<br><br>

```python
from dotenv import load_dotenv
from google import genai

# Cargamos nuestro archivo .env para inyectar la clave en memoria
load_dotenv()

# Inicializamos el cliente de Gemini
client = genai.Client()

# Paso 1: Definimos el System Prompt. Acá configuramos la Persona, el Tono y el Formato.
# Estas son las reglas inquebrantables del desarrollador que el modelo debe obedecer siempre.
system_instruction = """
You are a Senior AI Engineer. Your tone is direct and professional.
Always answer the user's question concisely and format your output using markdown bullet points.
"""

# Paso 2: Definimos el User Prompt. Esta es la entrada de datos o la tarea que envía nuestro usuario final.
user_query = "Explain the difference between System and User prompts."

# Paso 3: Estructuramos la anatomía del mensaje usando la API de Interactions.
# Inyectamos el System Prompt dentro de la configuración y enviamos la consulta del usuario.
interaction = client.interactions.create(
    model="gemini-3.5-flash",
    system_instruction=system_instruction,
    input=user_query
)

# Paso 4: La respuesta que nos devuelve el servidor asume automáticamente el rol de 'Assistant'.
# Extraemos el texto generado y lo imprimimos en consola.
print(interaction.output_text)
```
