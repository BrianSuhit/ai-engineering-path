Veamos cómo se ven estos artefactos en la práctica para un caso genérico: desarrollar una Landing Page para una cervecería artesanal *("The Rusty Keg")*.
En tu editor (como VS Code con Gemini, o Cursor), primero generás tu Constitución en una carpeta */specs*. Estos archivos dictarán las reglas para todo el proyecto:
```python
# specs/mission.md
**Mission:** Build a responsive landing page for "The Rusty Keg Brewery" to showcase our craft beers and location.
**Target Audience:** Craft beer enthusiasts and local tourists.
**MVP Scope:** A single-page site with a Hero section, Beer Menu, and a static Contact Map.
**Changelog Standard:** Use "Keep a Changelog" format and update it on every merged feature.

# specs/tech-stack.md
- **Frontend:** HTML5, CSS3, Vanilla JavaScript.
- **Styling:** Strictly pure CSS. Do not use Tailwind, Bootstrap, or any external framework.
```
Luego, pasás a la fase de Feature. Le pedís al agente que arme el documento de especificación para la primera tarea del Roadmap (la sección "Hero"). El agente genera este contrato:
```python
# specs/features/01-hero-section.md

## Plan
1. Create `index.html` with semantic tags.
2. Add a Hero section with a background image and a "View Our Beers" CTA.

## Requirements
- Use `<header>` and `<main>` tags for semantic structure.
- The CTA button must link to the `#menu` anchor.
- Strictly adhere to `specs/tech-stack.md` (No Tailwind).

## Validation
- The page loads without console errors.
- Clicking the CTA smoothly scrolls to the `menu` section.
```
Una vez que auditas y aprobás este archivo 01-hero-section.md, simplemente le decís a la IA en el chat de tu editor: "Implement the 01-hero-section feature exactly as specified". El agente leerá la Constitución para entender el proyecto, leerá el Feature Spec para entender la tarea, y generará código de producción predecible y blindado, listo para que lo valides.
