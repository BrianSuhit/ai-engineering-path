En el Machine Learning tradicional, evaluar un modelo era un problema que se resolvía con pura matemática. Si construías un clasificador para detectar si un correo era *Spam* o no, simplemente usabas métricas objetivas y estructuradas como la precisión (*Accuracy*) o el *F1-Score*. Tenías una respuesta correcta, el modelo te daba una predicción, y no había lugar para la subjetividad.
<br><br>
El gran dilema de la ingeniería de IA moderna es que **los LLMs generan texto abierto (*open-ended*)**. ¿Cómo evalúas matemáticamente si un resumen es "bueno", si un chatbot tiene "empatía" o si el agente programó algo con estilo limpio? Dos respuestas generadas por tu modelo pueden usar *tokens* completamente diferentes y, aun así, transmitir el mismo mensaje correcto de fondo. Una métrica tradicional de coincidencia exacta fracasa rotundamente acá.
<br><br>
Frente a esta dificultad, la industria reconoce unánimemente que **la evaluación humana es el estándar de oro (*Gold Standard*)**. Como los LLMs se construyen para interactuar con personas, la mejor forma de saber si un sistema funciona es que un humano lo lea y dictamine si sirve para el caso de uso.
<br><br>
Pero a nivel de arquitectura y despliegue, **depender de humanos no escala**. No podés pedirle a un equipo de expertos que lea 5.000 respuestas cada vez que modificás tu *System Prompt* o cambiás un parámetro en tu base de datos vectorial para ver si el sistema mejoró o empeoró.
<br><br>
Por lo tanto, la tarea del AI Engineer consiste en **construir tuberías de evaluación automáticas que intenten imitar o aproximar este juicio humano** para auditar que el modelo razone bien, use las herramientas correctas y no alucine. No existe una única métrica perfecta ni un estándar definitivo para todos los casos de uso.
