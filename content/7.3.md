Como ya establecimos, los LLMs son funciones matemáticas puras y sin estado (*stateless*); si les hacés una pregunta en el turno 1 y otra en el turno 2, por defecto no recuerdan absolutamente nada de lo que se habló antes. Para construir un sistema conversacional o un agente autónomo coherente, los *AI Engineers* tenemos que diseñar una arquitectura de memoria que le devuelva al modelo su "capacidad de recordar". En producción, dividimos esto en dos enfoques críticos: **Short-term Memory** y **Long-term Memory**.
<br><br>
La **Short-term Memory** (memoria a corto plazo) es el historial inmediato de la sesión activa. Consiste en adjuntar los mensajes recientes (lo que dijiste vos y lo que respondió el asistente) en cada nueva petición para que el modelo entienda el hilo de la charla. Por otro lado, si guardáramos cada chat de un usuario para siempre en el contexto activo, agotaríamos el límite de tokens y los costos explotarían. Por eso usamos **Long-term Memory** (memoria a largo plazo): almacenamos hechos clave, preferencias o interacciones pasadas del usuario en una base de datos externa (como PostgreSQL, ChromaDB o servicios especializados como Zep). Cuando el usuario vuelve a conectarse semanas después, el sistema recupera solo los datos relevantes de esa base de datos y los inyecta en el *System Prompt*, manteniendo la memoria RAM del modelo limpia.
<br><br>
Acá tenés cómo se ven ambos conceptos interactuando en código. Para el largo plazo, simulamos inyectar preferencias desde una base de datos. Para el corto plazo, aprovechamos el parámetro nativo **`previous_interaction_id`** de la nueva API de **Gemini**, que le indica al servidor que mantenga el estado de la sesión activa por nosotros, optimizando la memoria y bajando los costos:
<br><br>

```python
from dotenv import load_dotenv
from google import genai

# Cargamos el entorno e inicializamos el cliente
load_dotenv()
client = genai.Client()

# --- LONG-TERM MEMORY (Memoria Externa / A largo plazo) ---
# En producción, estos datos se recuperan de una base de datos externa para este usuario específico.
user_preferences = {"favorite_color": "#46778F", "profession": "AI Engineer"}
system_instruction = f"User context: {user_preferences}. Personalize your answers based on this context."

# --- SHORT-TERM MEMORY (Memoria de Sesión / A corto plazo) ---
# Turno 1: Arrancamos la sesión inyectando el estado a largo plazo.
interaction_1 = client.interactions.create(
    model="gemini-3.5-flash",
    system_instruction=system_instruction,
    input="Hi! Remind me, what is my favorite color?"
)
print("Turno 1:", interaction_1.output_text)

# Turno 2: Mantenemos la memoria a corto plazo usando 'previous_interaction_id'.
# El servidor de Google vincula automáticamente el historial reciente sin tener que reenviar los textos.
interaction_2 = client.interactions.create(
    model="gemini-3.5-flash",
    previous_interaction_id=interaction_1.id,
    input="Awesome. Can you suggest a secondary color that matches well with it?"
)
print("Turno 2:", interaction_2.output_text)

```
