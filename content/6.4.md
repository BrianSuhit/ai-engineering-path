En el mundo del software tradicional, el texto libre y "charlatán" de los LLMs es una pesadilla. Si queremos guardar el análisis de un modelo en una base de datos SQL o enviarlo a un *frontend*, no nos sirve que la IA responda con un amable *"¡Claro! Aquí tienes la información solicitada:"* antes de darnos los datos. Nuestro código espera variables estructuradas, precisión y previsibilidad.
<br><br>
Para solucionar esto utilizamos **JSON Prompting**. Aunque podemos rogarle al modelo en el *prompt* que solo nos devuelva un JSON, la arquitectura moderna nos permite forzar este comportamiento a nivel del motor. Utilizando **Structured Outputs** de forma nativa, bloqueamos al modelo para que no pueda generar nada más que un objeto JSON válido.
<br><br>
La mejor forma de implementar esto en Python es combinando la API del modelo con **Pydantic**, una librería estándar de validación de datos. Nosotros definimos un esquema estricto (qué variables queremos y de qué tipo son), se lo inyectamos en la configuración, y el servidor remoto se encarga de obligar al modelo a rellenar esa estructura exacta, eliminando las alucinaciones de formato.
<br><br>
Acá tenés cómo se ve esta arquitectura defensiva en código usando Gemini y Pydantic:
<br><br>

```python
from dotenv import load_dotenv
from google import genai
from pydantic import BaseModel

load_dotenv()
client = genai.Client()

# Paso 1: Definimos la estructura de datos que nuestro software necesita usando Pydantic.
# El modelo estará obligado a devolver exactamente estas llaves y estos tipos de datos.
class SentimentAnalysis(BaseModel):
    sentiment: str
    confidence: float
    key_phrases: list[str]

# Paso 2: Creamos las instrucciones y la consulta del usuario
system_instruction = "You are a data extraction assistant. Analyze the text and return ONLY a valid JSON."
user_query = "Review: The battery life is decent, but the screen scratches easily."

# Paso 3: Ejecutamos la inferencia forzando la salida estructurada.
# Pasamos nuestro esquema de Pydantic directamente en la configuración de la API.
interaction = client.interactions.create(
    model="gemini-3.5-flash",
    system_instruction=system_instruction,
    input=user_query,
    response_format={
        "type": "text",
        "mime_type": "application/json",
        "schema": SentimentAnalysis.model_json_schema()  # <- Convierte Pydantic a JSON Schema crudo
    }
)

# Paso 4: Validamos el JSON y parseamos la respuesta a un objeto tipado de Python.
# Si el LLM rompió el contrato de datos, Pydantic lanzará un ValidationError controlabe.
valid_result = SentimentAnalysis.model_validate_json(interaction.output_text)

# Paso 5: Consumimos los datos de forma segura en nuestra lógica de negocio.
print(f"Sentimiento: {valid_result.sentiment}")
print(f"Confianza: {valid_result.confidence}")
print(f"Frases clave: {valid_result.key_phrases}")
```
