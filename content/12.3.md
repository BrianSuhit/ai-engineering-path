A nivel de arquitectura, un servidor MCP se comunica con el mundo exterior exponiendo tres capacidades o primitivas fundamentales. Estas primitivas definen exactamente qué tipo de información y poder de ejecución le estamos otorgando a los modelos de Inteligencia Artificial para que interactúen con nuestra infraestructura.
<br><br>
Las **Tools** (Herramientas) son funciones ejecutables que el modelo de IA decide invocar para realizar acciones que suelen tener efectos secundarios en el entorno, como consultar una API del clima, enviar un correo o escribir en una base de datos. Por el contrario, los **Resources** (Recursos) son fuentes de datos estrictamente de solo lectura que el cliente puede consultar mediante una URI específica (como si fuera una ruta de archivo local) para inyectar contexto seguro sin riesgo de modificar nada.
<br><br>
Finalmente, los **Prompts** son plantillas de instrucciones o flujos de conversación predefinidos que el propio servidor MCP provee. Esto es súper útil para estandarizar comportamientos (por ejemplo, definir un rol estricto de revisor de código) directamente desde el servidor, asegurando que cualquier modelo que se conecte sepa exactamente cómo formatear su análisis sin tener que programarlo en el cliente.
