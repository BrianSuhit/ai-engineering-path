En la ingeniería de IA, los LLMs presentan un problema fundamental: son inherentemente no deterministas. Si bien los desarrolladores aplican técnicas como *Prompt Engineering*, *Fine-Tuning*, arquitecturas RAG o modelos alineados con RLHF, la realidad es que ninguna de estas estrategias por sí sola puede eliminar por completo la variabilidad y la imprevisibilidad de los resultados. Esto crea un obstáculo crítico para llevar aplicaciones a producción, especialmente en industrias reguladas.
<br><br>
Para resolver esta imprevisibilidad nacen los **Guardrails**. A nivel de sistemas, un *guardrail* es un mecanismo de seguridad y una herramienta de validación integrada en la aplicación que garantiza en tiempo real (*runtime*) que el sistema opere dentro de límites predefinidos. Actúan como un marco protector que previene resultados no deseados y alinea el comportamiento del modelo con las expectativas exactas del desarrollador.
<br><br>
En lugar de confiar ciegamente en que el LLM hará lo correcto, el flujo de datos cambia para incluir una verificación explícita. El sistema envía la información a través de una barrera de validación antes de que el *prompt* llegue al modelo (*Input Guard*), o intercepta la respuesta generada antes de devolvérsela al usuario (*Output Guard*).
<br><br>
El corazón arquitectónico de cualquier *guardrail* es un componente llamado **Validator**. Un *validator* es simplemente una función de código que toma como entrada el *user prompt* o la respuesta del LLM y verifica matemáticamente o por lógica dura si ese texto cumple con una regla predefinida.
<br><br>
La complejidad de este *validator* depende puramente de tu caso de uso. Puede ser tan simple como usar una expresión regular (regex) en Python para bloquear números de teléfono y evitar la fuga de información personal (PII). O puede ser tan avanzado como usar un modelo de *Natural Language Inference* (NLI) para verificar si la respuesta generada está estrictamente fundamentada en el texto recuperado de tu base de datos vectorial, bloqueando así las alucinaciones en un sistema RAG.
