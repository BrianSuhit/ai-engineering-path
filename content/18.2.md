A medida que los proyectos escalan, el Vibe Coding colapsa. Acá pasás al nivel intermedio: dejás de pedir "aplicaciones mágicas" y empezás a usar la IA como un copiloto táctico interactuando directamente con el chat de tu editor (como Gemini en VS Code o Copilot Chat). En esta etapa, usás el asistente para ejecutar tareas quirúrgicas: aislar un bug, generar unit tests, o auditar Technical Debt (esa base de código espagueti y sin documentar que heredaste y nadie quiere tocar).
<br><br>
Para que el chat de tu editor actúe de forma predecible y no rompa tu arquitectura, tenés que abandonar las instrucciones vagas y usar **Prompt Templates** estructurados. Los desarrolladores suelen tener estos moldes guardados en un archivo de texto para copiarlos y pegarlos rápidamente en la barra del chat junto con su código. La anatomía de un prompt sólido se divide en tres componentes: **Priming** (preparar a la IA asumiendo un rol experto), **Question** (el fragmento de código crudo a evaluar) y el **Decorator** (las restricciones estrictas sobre cómo querés que te devuelva la respuesta en la interfaz).
<br><br>
El mayor riesgo al hacer pair programming conversacional es confiar ciegamente. El modelo puede arrojar código refactorizado que parece súper eficiente, pero a veces alucina el comportamiento de las herramientas (como sugerir métodos que no existen en una librería). El copiloto te da velocidad para entender código viejo, pero vos sos el único responsable de auditar la lógica antes de aceptarla en tu proyecto.
<br><br>
```python
# Pair Programming Prompt Template (Para pegar en el Chat del IDE)

[Priming]
You are an expert at writing clear, concise, and secure Python code.

[Question]
Review the following legacy code block. It implements a doubly linked list but has a hidden bug that causes null pointer exceptions. 
`[INSERTA TU CÓDIGO HEREDADO ACÁ]`

[Decorator]
Find the bug, suggest a fix, and insert comments for each line of code explaining the new logic. Output only the refactored code without plain text explanations.
```
